# Ключи API и настройки

От тебя потребуются **ключи и доступы** к сервисам из ТЗ. Без них система будет работать локально (лиды, диалог, очередь), но без реальной отправки в WhatsApp, без Calendly и без LLM.

## Что нужно получить и куда подставить

### 1. amoCRM (Kommo)

- Зарегистрировать приложение (интеграцию) в amoCRM и получить **OAuth2 access token**.
- В `.env`:
  - `AMOCRM_SUBDOMAIN` — поддомен (например, `mycompany` для mycompany.amocrm.ru).
  - `AMOCRM_ACCESS_TOKEN` — токен доступа.

Используется для: вебхуки лидов, Chats API (отправка/приём сообщений WhatCRM), обновление сделок и задач.

### 2. WhatCRM / WhatsApp

- WhatCRM подключается как виджет в amoCRM; каждый номер — чат-канал в amoCRM.
- Отправка и приём сообщений идут **через amoCRM Chats API** (достаточно `AMOCRM_ACCESS_TOKEN`).
- Дополнительно: если нужна проверка статуса подключения номеров через WhatCRM API — ключ `WHATCRM_API_KEY` в `.env` (опционально).

### 3. Calendly

- Личный или организационный **API token** в Calendly.
- В `.env`: `CALENDLY_API_TOKEN`, при необходимости `CALENDLY_EVENT_TYPE_UUID` (UUID типа встречи для Zoom).

Нужно для: получения слотов, создания встреч, вебхуков invitee.created / invitee.canceled.

### 4. LLM (ответы бота)

- Сейчас заложена поддержка **OpenAI**: в `.env задаётся `OPENAI_API_KEY`.
- Для казахского/шала-казахского по ТЗ — выбрать модель, которая их поддерживает (например, GPT-4).

Когда ключ будет — он подставляется в `.env`; вызов LLM в коде можно добавить в `DialogService` или отдельный `LlmService`.

### 5. Admin Panel

- `JWT_SECRET` — любая длинная случайная строка (для подписи JWT при логине в админку).

### 6. Уведомления (по желанию)

- **SLA / руководитель**: телефон в WhatsApp для уведомлений о просроченном handoff — в ТЗ и в конфиге как `SUPERVISOR_WHATSAPP`.
- **Алерты в Telegram**: `TELEGRAM_BOT_TOKEN` и `TELEGRAM_ALERT_CHAT_ID` в `.env` (если будешь включать алерты в Telegram).

---

## Итого: минимум для «всё включено»

| Переменная             | Где взять                    | Обязательно для прода |
|------------------------|-----------------------------|------------------------|
| `AMOCRM_SUBDOMAIN`     | Твой поддомен amoCRM        | да                    |
| `AMOCRM_ACCESS_TOKEN`  | OAuth2 в amoCRM             | да                    |
| `CALENDLY_API_TOKEN`   | Calendly → API token        | да (для Zoom)         |
| `OPENAI_API_KEY`       | OpenAI (или аналог)        | да (для ответов бота) |
| `JWT_SECRET`           | Придумать сам               | да (для админки)      |
| `DATABASE_URL`         | Твой Postgres на сервере    | да                    |
| `REDIS_HOST` / `REDIS_PORT` | Твой Redis на сервере | да                    |

Остальные переменные в `.env.example` — опциональны или с подсказками в комментариях.

После того как ключи будут — подставь их в `.env` на сервере (или локально для теста). Код уже рассчитан на чтение этих переменных окружения.
