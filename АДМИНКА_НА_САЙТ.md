# Админка на https://chatbot1.vps.webdock.cloud/

Чтобы по этому адресу открывалась админка и она ходила в API бота, сделай два шага.

---

## 1. Собрать админку и положить в корень сайта (на сервере)

В SSH по одной команде (каждую строку — Enter):

```bash
cd /var/www/ai-sales-bot
git pull origin main
cd admin-frontend
npm install
npm run build
```

Потом заменить содержимое веб-корня на сборку админки:

```bash
sudo rm -f /var/www/html/index.php
sudo cp -r /var/www/ai-sales-bot/admin-frontend/dist/* /var/www/html/
```

Проверь, что в корне появились файлы:

```bash
ls -la /var/www/html/
```

Должны быть хотя бы `index.html` и папка `assets`.

---

## 2. Настроить nginx: запросы /api отправлять на бота

Сейчас запросы к `https://chatbot1.vps.webdock.cloud/api/...` должны уходить на приложение (порт 3000). Если этого ещё нет:

Найди конфиг nginx для этого сайта (часто в `/etc/nginx/sites-available/` или в панели Webdock). В блок `server` для `chatbot1.vps.webdock.cloud` добавь:

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:3000/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

Перезагрузи nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

После этого:
- открывается **https://chatbot1.vps.webdock.cloud/** — админка;
- запросы к **/api/** уходят на бота, конфиг в админке подгружается.

Если «ничего не происходит» — пришли вывод команд `ls -la /var/www/html/` и (если есть) ошибку из браузера (F12 → вкладка Network или Console).
