# Деплой: заливка и команды на сервере

**После любых изменений в коде:**

1. Запустить батник **залить на GitHub** (у себя).
2. Зайти на сервер и выполнить команды **в терминале на сервере**.

---

## 1. Заливка на GitHub (у себя на компьютере)

Запусти батник из папки проекта:

```
1_залить_на_гитхаб.bat
```

Двойной клик — он сделает add, commit и push в GitHub. Всё.

---

## 2. Команды на сервере (в терминале на сервере)

Зайди на сервер (панель хостинга, терминал Webdock и т.п.) и открой терминал. Путь к проекту подставь свой (например `/var/www/ai-sales-bot`).

### Вариант A: один скрипт (если есть deploy.sh)

```bash
cd /var/www/ai-sales-bot
chmod +x deploy.sh
./deploy.sh
```

Скрипт сделает: git pull, npm ci, prisma generate, prisma migrate deploy, npm run build, pm2 restart.

---

### Вариант B: команды по шагам

```bash
cd /var/www/ai-sales-bot

git pull origin main

npm install
npx prisma generate
npx prisma migrate deploy
npm run build

pm2 restart ai-sales-bot --update-env
```

Если приложение запускается не через PM2 — перезапусти свой сервис вместо последней строки.

---

### Сборка админки (если раздаётся с этого сервера)

После бэкенда:

```bash
cd /var/www/ai-sales-bot/admin-frontend
npm install
npm run build
```

Потом скопировать содержимое `dist` в нужную папку для сайта.

---

## Кратко

| Шаг | Где | Что сделать |
|-----|-----|-------------|
| 1 | У себя | Запустить **`1_залить_на_гитхаб.bat`** |
| 2 | Сервер (терминал) | `cd /var/www/ai-sales-bot` → `./deploy.sh` **или** команды по шагам (git pull, npm install, prisma migrate deploy, npm run build, pm2 restart) |

---

## Первый раз на сервере (если ещё не разворачивал)

1. Клонировать: `git clone https://github.com/mirosnikovevpatij-design/ai-sales-bot.git` в нужную папку.
2. Создать `.env`: `cp .env.example .env` и заполнить по **КЛЮЧИ_И_НАСТРОЙКИ.md**.
3. Установка и запуск: `npm install`, `npx prisma generate`, `npx prisma migrate deploy`, `npm run build`, затем `pm2 start dist/main.js --name ai-sales-bot`.
4. Настроить nginx: `/api` → `http://127.0.0.1:3000`. Подробнее — **ДЕПЛОЙ_НА_СЕРВЕР.md**.
