# Вебхуки amoCRM — настройка и формат данных

## Какой URL указывать в amoCRM

В настройках вебхуков amoCRM (раздел «Вебхуки» в интеграциях) нужно указать **один адрес** — на него amoCRM будет отправлять все выбранные события.

**Рекомендуемый URL:**

```
https://chatbot1.vps.webdock.cloud/api/webhooks/amo
```

- Домен `chatbot1.vps.webdock.cloud` — ваш сервер (админка открывается по этому же адресу).
- Путь `/api/webhooks/amo` — единая точка входа для всех событий (сделки, статусы и т.д.).

Итоговая ссылка для поля **destination**: `https://chatbot1.vps.webdock.cloud/api/webhooks/amo`

Отдельный URL для админки не нужен: админка — это тот же хост (тот же домен), другой путь (например `/` или `/admin`). Вебхуки принимает только бэкенд по пути `/api/webhooks/amo`.

---

## Подписка на события (settings)

Для работы бота нужно подписаться минимум на:

| Событие      | Описание                    |
|-------------|-----------------------------|
| **add_lead**   | Добавлена сделка            |
| **status_lead**| У сделки сменился статус     |

При регистрации вебхука в запросе передайте, например:

```json
{
  "destination": "https://chatbot1.vps.webdock.cloud/api/webhooks/amo",
  "settings": ["add_lead", "status_lead"]
}
```

Так бот будет получать и новые сделки, и переводы в «Неразобранное» и другие статусы.

---

## Формат запроса от amoCRM

amoCRM шлёт на ваш URL **POST** с телом в формате JSON. Структура может быть такой (вариант с массивом сделок):

### Событие add_lead

В теле может быть объект с массивом добавленных сделок, например:

```json
{
  "account": { "id": 123456 },
  "leads": {
    "add": [
      {
        "id": 12345,
        "name": "Новая сделка",
        "status_id": 123,
        "pipeline_id": 1,
        "responsible_user_id": 456,
        "created_at": 1700000000,
        "updated_at": 1700000000,
        "_embedded": {
          "contacts": [
            {
              "id": 789,
              "phone": "+77001234567"
            }
          ]
        }
      }
    ]
  }
}
```

### Событие status_lead

При смене статуса приходит что-то вроде:

```json
{
  "account": { "id": 123456 },
  "leads": {
    "status": [
      {
        "id": 12345,
        "status_id": 456,
        "old_status_id": 123,
        "pipeline_id": 1
      }
    ]
  }
}
```

Точную схему по полям смотрите в официальной документации amoCRM (раздел «Формат получаемых хуков»).

---

## Что делает наш бэкенд

- Принимает **POST** на `/api/webhooks/amo`.
- Разбирает тело:
  - если есть **leads.add** или **leads.status** — обрабатывает каждую сделку из массива;
  - для каждой сделки берёт **id** (это `deal_id`), по возможности телефон из вложенных контактов или из полей сделки.
- По паре deal_id + телефон создаёт или обновляет сессию лида (стоп-лист, очередь первого сообщения и т.д.), как в ТЗ.

Если в payload нет телефона, бот может достраивать контакт через API amoCRM (GET сделки/контакта), если настроены токен и вызовы к amoCRM.

---

## Проверка

1. В amoCRM: Настройки → Интеграции → Вебхуки → подписать на `add_lead` и `status_lead` с destination = `https://chatbot1.vps.webdock.cloud/api/webhooks/amo`.
2. Создать тестовую сделку с контактом и телефоном — на сервер должен прийти POST, в логах бэкенда — обработка, в БД — запись по лиду (при наличии телефона и при отсутствии в стоп-листе).
